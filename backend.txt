2.1 GRADING
───────────
  getGrade(pct, gradingScale)
    → Loop gradingScale array (sorted by min_percentage DESC)
    → Return grade where pct >= min_percentage AND pct <= max_percentage
    → Fallback to hardcoded: A+(90-100), A(80-89), B(70-79), C(60-69), D(50-59), F(0-49)
    → NEVER use calculateGrade() directly — always pass through getGrade()

  getGradeClass(pct)
    → Returns CSS class: grade-Ap|grade-A|grade-B|grade-C|grade-D|grade-F

  Pass/Fail threshold:
    → Always read from: parseFloat(state.schoolSettings?.pass_mark || 50)
    → NEVER hardcode >= 50 anywhere in the codebase

2.2 MARKS COMPLETION RATE (Dashboard stat card)
────────────────────────────────────────────────
  Formula:
    totalPossible = SUM over each term-assessment of:
                      count of Active students in that assessment's class
    totalFilled   = SUM over each term-assessment of:
                      IF assessment.is_locked → add class student count (fully done)
                      ELSE → count of marks records for that assessment
    completionPct = totalFilled / totalPossible × 100

  Rules:
    · Only current-term assessments
    · Locked assessments = 100% complete
    · Absent marks count as entered (teacher made a decision)
    · If totalPossible === 0 → show 0%

2.3 PRE-MIDTERM CALCULATION (Primary)
──────────────────────────────────────
  For each assessment in pre_midterm phase:
    raw_pct = score / max_marks × 100
  Average of all pre-midterm raw_pcts = preMidtermPct

  calcPreMidtermPrimary(scores[], maxes[]):
    total_score = sum(scores)
    total_max   = sum(maxes)
    return total_max > 0 ? (total_score / total_max) × 100 : 0

2.4 PRE-MIDTERM CALCULATION (Nursery)
──────────────────────────────────────
  calcPreMidtermNursery(scores[]):
    → Simple average of scores (no max scaling)
    return scores.length > 0 ? sum(scores) / scores.length : 0

2.5 MG (Moyenne Générale — Continuous Assessment)
──────────────────────────────────────────────────
  calcMG(scores[], maxes[], mgMax):
    avgPct = sum(scores) / sum(maxes) × 100    [if sum(maxes) > 0]
    return (avgPct / 100) × mgMax

  Example: scores=[18,22], maxes=[25,25], mgMax=30
    avgPct = 40/50 × 100 = 80%
    MG = 80/100 × 30 = 24/30

2.6 EXAM (EX) COMPONENT
─────────────────────────
  calcEX(scores[], maxes[], exMax):
    Same formula as MG but for exam assessments
    avgPct = sum(scores) / sum(maxes) × 100
    return (avgPct / 100) × exMax

2.7 POST-MIDTERM SUBJECT TOTAL
────────────────────────────────
  calcSubjectPostMidterm(subject, assessments, marks, studentId):
    mgAssessments  = assessments.filter(phase='pre_midterm' OR type!='Final Exam')
    exAssessments  = assessments.filter(type='Final Exam' OR type='Exam')
    mg = calcMG(mgScores, mgMaxes, subject.mg_max)
    ex = calcEX(exScores, exMaxes, subject.ex_max)
    total = mg + ex
    maxTotal = subject.mg_max + subject.ex_max
    pct = total / maxTotal × 100
    return { mg, ex, total, maxTotal, pct, grade: getGrade(pct) }

2.8 ANNUAL SUBJECT TOTAL (Full-year transcript)
────────────────────────────────────────────────
  calcSubjectAnnual(subject, allTermAssessments, allMarks, studentId):
    Same as post-midterm but spans ALL terms in the academic year
    Uses same MG + EX formula across the full year's assessments

2.9 STUDENT RANKING
────────────────────
  rankStudents(students[]):
    → Sort by percentage DESC, then by name ASC for ties
    → Olympic ranking: if 2 students share rank 1, next rank is 3
    → Return [{...student, rank, total, pct}]

  calculateStudentRank(studentId, classId):
    → Get all active students in that class
    → Calculate each student's term percentage
    → Sort, rank, return "X of Y" string for the specific student

  calculateStudentRankFair(studentId, classId, termsToProcess, ...):
    → Multi-term weighted rank
    → Handles nursery vs primary level differences
    → Used for year-end report cards

2.10 ATTENDANCE RATE
─────────────────────
  attendanceRate(studentId, dateFrom, dateTo):
    totalDays   = count of attendance records in range for that student
    presentDays = count where status IN ('Present', 'Late')
    return totalDays > 0 ? (presentDays / totalDays) × 100 : null

  Status for at-risk alert: attendanceRate < 75%

3.1 STUDENT BALANCE CALCULATION
────────────────────────────────
  studentFeeBalance(studentId):
    fees       = student_fees WHERE student_id=X AND is_waived=false AND is_credit=false
    credits    = student_fees WHERE student_id=X AND is_credit=true
    totalDue   = sum(fees.amount) - sum(fees.waiver_amount)
    totalPaid  = sum(payments via payment_allocations)
    creditAmt  = sum(credits.amount)
    balance    = totalDue - totalPaid - creditAmt
    return { totalDue, totalPaid, balance, credit: creditAmt,
             pct: totalDue>0 ? (totalPaid/totalDue)×100 : 100,
             hasCredit: creditAmt > 0 }

  Rules:
    · Waived fees: excluded from totalDue
    · Credits: reduce what is owed (overpayments become credits)
    · Archived fees: excluded from current balance

3.2 FEE RESET AUTOMATION
────────────────────────
  Triggered: daily check at boot and every 24h

  Monthly fees:  apply on 1st of each month
  Termly fees:   apply day after term.start_date
  Annual fees:   apply day after academic_year.start_date
  One-time fees: applied manually only

  applyFeesByFrequency(frequency):
    For each active student:
      For each fee_category WHERE frequency matches:
        Check if student_fees record already exists for this period
        If not → get fee_amounts for student's class
        Apply sibling discount if family.discount_pct > 0
        Apply any active recurring waivers
        Insert student_fees record

  archivePreviousYearFees():
    Mark student_fees.is_archived = true
    WHERE academic_year_id != current AND is_paid = false

3.3 PAYMENT RECORDING
──────────────────────
  recordPayment(studentId, amount, method, reference, recordedBy, recordedByRole):
    1. Validate: amount > 0, student exists, student is Active
    2. Generate receipt_number: "R-" + year + "-" + padded sequence
    3. INSERT into payments
    4. Allocate payment to oldest unpaid fees first (FIFO):
       - Get student_fees WHERE is_paid=false ORDER BY due_date ASC
       - Allocate payment amount across fees until exhausted
       - Mark fee.is_paid=true when fully covered
       - If payment > total owed → create credit record in student_fees (is_credit=true)
    5. Dispatch notification:
       - IF recordedByRole === 'admin' → notify accountant
       - IF recordedByRole === 'accountant' → notify admin
       - notification.type = 'payment', resource_id = payment.id
    6. Log activity

3.4 PAYMENT REVERSAL
─────────────────────
  reversePayment(paymentId, reason, reversedBy):
    1. Validate payment exists and is not already reversed
    2. SET payments.is_reversed=true, reversed_at=now(), reversal_reason=reason
    3. De-allocate: UPDATE student_fees paid_amount -= allocated_amount
    4. Remove payment_allocations for this payment
    5. Remove credit records created by this payment
    6. Notify admin + accountant
    7. Log activity

3.5 FEE WAIVER
───────────────
  applyWaiver(studentFeeId, waiverType, amount, reason):
    Types:
      'full'       → is_waived=true, waiver_amount=fee.amount
      'percentage' → waiver_amount = fee.amount × pct / 100
      'custom'     → waiver_amount = specified amount (cannot exceed fee.amount)
    
    If fee already partially paid → waiver applies to remaining balance only
    Log activity with reason

3.6 OVERDUE CLASSIFICATION
───────────────────────────
  Days overdue = today - fee.due_date (where due_date is in the past and not paid)
  
  Critical  = days > 44
  High      = days 30-44
  Medium    = days 14-29
  Recent    = days 7-13
  Not yet   = days <= 6 (not overdue)

3.7 COLLECTION RATE
────────────────────
  collectionRate(classId, termId):
    expected  = sum(student_fees.amount) for active students in class this term
    collected = sum(payments.amount) allocated to those fees
    return expected > 0 ? (collected / expected) × 100 : 0

3.8 SIBLING DISCOUNT
─────────────────────
  If student.family_id is set AND families.discount_pct > 0:
    Discount applies to all fee categories marked as discountable
    discounted_amount = fee_amount × (1 - discount_pct / 100)
    Applied automatically when fees are generated for that student

SECTION 4 — NOTIFICATION DISPATCH RULES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Every significant action dispatches a notification via notifyAction():

  ACTION                  → NOTIFY ROLES       ICON  TYPE
  ─────────────────────────────────────────────────────────
  Admin records payment   → accountant         💰    payment
  Accountant records pay  → admin              💰    payment
  Teacher saves marks     → admin              📝    marks
  Admin edits marks       → teacher (owner)    ✏️    marks_edit
  Marks locked            → admin              🔒    marks
  Student enrolled        → admin              🎓    student
  Fee waiver applied      → admin+accountant   🎁    finance
  Payment reversed        → admin+accountant   ↩️    payment
  Backup created          → admin              💾    system
  Attendance not recorded → class teacher      📋    attendance (daily reminder)

  notifyAction(action, details, targetRoles):
    INSERT INTO notifications:
      title        = icon + " " + humanised(action)
      message      = details.message
      type         = categorise(action)
      recipient_role = targetRoles[0] (or 'all')
      recorded_by  = currentUser.name
      resource_id  = details.resource_id
      is_read      = false

  Polling: notifications are polled every 60 seconds
  Bell badge: count of is_read=false for current user's role

SECTION 6 — MARKS ENTRY VALIDATION RULES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  On every keystroke in a score input:
    1. Parse raw input as float
    2. If NaN → show inline popup: "❌ Invalid value" — offer Clear
    3. If < 0  → show inline popup: "⚠️ Below zero" — offer Auto-correct to 0 | Manual fix | Clear
    4. If > max_marks → show inline popup: "⚠️ Exceeds max" — offer Auto-correct to max | Manual fix | Clear
    5. If valid → commit score, update % and grade badge live, clear popup

  Popup has 3 options (non-numeric gets only Clear):
    A. ✅ Auto-correct to [0 or max]   → sets value, closes popup
    B. ✏️ Let me type the correct value → closes popup, refocuses+selects input
    C. 🗑️ Clear                        → empties field, closes popup

  Save gate:
    BLOCK saving if ANY popup is open (errorPopups.length > 0)
    BLOCK saving if ANY student input is empty AND not marked absent
    Both conditions produce a specific toast + scroll to problem

  After save:
    → dispatch notification to admin: marks_saved action
    → include: teacher name, assessment name, subject, class, count saved

  Summary screen (before final save):
    → Shows: entered count, absent count, average, pass rate
    → ASCII grade distribution chart
    → Per-student score grid
    → User must click "Confirm & Save" to actually write to DB

SECTION 8 — AUTHENTICATION & SESSION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Session stored in localStorage: keys elf_user (JSON) + elf_expiry (timestamp)
  Session duration: 1 hour (rolling — reset on any click/keydown/mousemove)
  Idle warning: 25 minutes of inactivity
  Idle logout: 30 minutes of inactivity
  Max login attempts: 5 → 15-minute lockout
  Session watcher: checks every 60 seconds

  Admin login: compare password vs school_settings.admin_password
  Teacher/Accountant login: query teachers table by username+role, compare password

  Biometric (WebAuthn):
    Registration: navigator.credentials.create() with platform authenticator
    Assertion: navigator.credentials.get()
    Credential ID stored in localStorage as biometric_{userId}

  Password change:
    Admin → update school_settings.admin_password
    Teacher → update teachers.password
    Requires current password verification
    Forces logout after change

SECTION 9 — API LAYER (Supabase REST)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Base URL: SUPABASE_URL + '/rest/v1/'
  Auth header: apikey: SUPABASE_KEY, Authorization: Bearer SUPABASE_KEY
  Content-Type: application/json
  Prefer: return=representation (on POST/PATCH to get back the row)

  Core wrappers:
    api(path, method, body)          → raw fetch with error handling
    get(table, params)               → GET with auto-pagination for marks/assessments
    getAll(table, filters)           → high-level with 50000 limit
    getById(table, id)               → single row by id
    insert(table, data)              → POST, returns first element
    update(table, id, data)          → PATCH by id
    delete_(table, id)               → DELETE by id
    apiRequest(endpoint, method, body) → cleaner wrapper returning {success, data, error}

  Pagination:
    marks and assessments: auto-paginate in 1000-row batches
    all other tables: limit=50000 single request

  Offline handling:
    Network errors → call notifyOffline() (de-duped toast max once per 10s)
    Queue marks in IndexedDB for offline entry
    Sync on reconnect via syncOfflineMarks()

  Error classification:
    Network error → 'OFFLINE: Could not reach the server'
    4xx → Supabase API error with detail
    5xx → Server error

  Auto-archive trigger: student.status = 'Inactive' for > 30 days
  OR: student has no marks, no attendance, no payments for > 90 days
  Archived students: is_archived=true, archived_at=now()
  Archived students excluded from all calculations
  Restore: is_archived=false, archived_at=null

  Backup format: JSON file containing all state tables
  Filename: ELF_Backup_YYYY-MM-DD_HH-MM.json

  Manual backup: triggered from Settings > Backup & Restore
