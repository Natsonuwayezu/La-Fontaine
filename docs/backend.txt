## 3.1 AUTHENTICATION

Rule: Admin logs in with only password (no username).
  - Password stored in school_settings WHERE key = 'admin_password'
  - On login: compare input with stored value
  - On success: save session { role: 'admin', id: null } to localStorage

Rule: Teacher/Accountant logs in with username + password.
  - Query: SELECT * FROM teachers WHERE username = $1 AND password = $2
  - On success: save session { role: teacher.role, id: teacher.id } to localStorage

Rule: Session persists across page refresh (from localStorage).
Rule: Session expires after 40 minutes of inactivity.
Rule: Max 4 failed login attempts → show lockout message.

---

## 3.2 ROLE-BASED ACCESS CONTROL

Admin:        Access to ALL modules
Accountant:   ONLY Finance modules (fee structure, payments, reports, overdue, waivers, receipts)
              CANNOT see: Academics, Students, Staff, Settings
Teacher:      ONLY Academics + own student list
              CANNOT see: Finance, Settings (except own profile), Staff management

Sidebar builds differently per role.
API calls must be filtered by role server-side (via Supabase RLS or custom checks).

---

## 3.3 PHASE DETECTION

The system operates in two phases per term:

  PRE_MIDTERM:  Today < term.midterm_date
  POST_MIDTERM: Today >= term.midterm_date

This affects:
- What subjects appear in Class Register (post_midterm_only subjects hidden pre-midterm)
- What assessment types are available in Marks Entry
- What columns appear in Class Register tables
- Report card format (Mid-Term vs End of Term)
- Labels on topbar

## 3.6 MARKS RULES

Validation:
  - Score cannot be negative → set to 0
  - Score cannot exceed assessment.max_marks → cap at max_marks
  - Decimals → round to 1 decimal place
  - Empty score → skip (do not save)

Auto-behavior:
  - For post_midterm_only subjects: when teacher saves EX marks,
    system saves a duplicate as MG marks automatically
  - If assessment is locked → reject any save attempt

Offline support:

  - Show "X marks pending sync" banner

## 3.8 FEE MANAGEMENT RULES

### When Admin creates a new fee category:
  1. INSERT into fee_categories
  2. For each active class:
     - If fee_amounts entry doesn't exist → prompt admin to set amount per class
  3. After amounts set → FOR EACH active student:
     INSERT into student_fees {
       student_id, fee_category_id,
       academic_year_id = current_year,
       term_id = current_term (if termly) or NULL (if annual/one-time),
       amount = fee_amounts[student.class_id],
       paid_amount = 0,
       is_paid = FALSE,
       due_date = current_term.end_date
     }
  4. Log action to system_logs

### When fee resets (automated):
  Monthly (runs 1st of each month):
    FOR EACH fee_category WHERE reset_frequency = 'monthly' AND is_active = TRUE:
      FOR EACH student WHERE status = 'Active':
        INSERT new student_fees record

  Termly (runs on term.end_date):
    FOR EACH fee_category WHERE reset_frequency = 'termly' AND is_active = TRUE:
      FOR EACH student WHERE status = 'Active':
        INSERT new student_fees record for the new term

  Annual (runs on academic_year.end_date):
    FOR EACH fee_category WHERE reset_frequency = 'annual' AND is_active = TRUE:
      FOR EACH student WHERE status = 'Active':
        INSERT new student_fees record for new year

### When payment is recorded:
  1. Generate receipt_number: RCP-YYYYMMDD-NNN
  2. INSERT into payments
  3. Allocate payment using FIFO (oldest unpaid fees first):
     a. SELECT student_fees WHERE student_id = X AND is_paid = FALSE
        ORDER BY created_at ASC
     b. For each fee:
        - allocate = MIN(remaining_payment, fee.amount - fee.paid_amount)
        - INSERT into payment_allocations (payment_id, student_fee_id, allocate)
        - UPDATE student_fees SET paid_amount += allocate
        - IF paid_amount >= amount → SET is_paid = TRUE
        - remaining_payment -= allocate
        - STOP when remaining_payment = 0
     c. If payment > all remaining fees:
        - UPDATE student_credit_balance SET credit_amount += leftover
  4. Log action to system_logs
  5. CREATE notification for Admin (if Accountant recorded) or vice versa

### Credit balance rules:
  - Student can have a positive credit balance (overpaid)
  - When a new fee is added for a student with credit:
    → Auto-deduct from credit_amount first
    → If credit >= new fee → mark fee as paid immediately
    → If credit < new fee → partial pay from credit, remaining is due
  - Admin can manually adjust credit (add/deduct/refund)
  - Credit is per student (not per family)

### Payment reversal:
  1. SET payments.is_reversed = TRUE, reversed_at, reversed_by, reversal_reason
  2. DELETE payment_allocations for this payment
  3. Recalculate student_fees.paid_amount and is_paid for affected fees
  4. Adjust student_credit_balance if credit was involved
  5. Log action

### Fee waiver rules:
  - Admin or Accountant can waive a fee for a student
  - SET student_fees.is_waived = TRUE, waiver_reason, waiver_by
  - INSERT into fee_waivers
  - Waived fees are excluded from Total Fees calculation
  - Waived fees are NOT counted as Paid either
  - Formula: Balance = (Total Fees - Waived) - Paid

## 3.9 NOTIFICATION RULES

System auto-creates notifications when:
  - Payment recorded (to opposite role: admin→accountant, accountant→admin)
  - New student enrolled (to admin)
  - Assessment locked (to teacher who owns that class/subject)
  - Marks entry overdue (to teacher and admin)
  - Overdue payment (to accountant)
  - Monthly/termly fee reset completed (to admin)
  - System backup completed (to admin)

Admin can manually send announcements:
  - To: all users, teachers only, accountants only, or specific user
  - Type: general, urgent, event
  - Creates notification records for each recipient

Bell counter = count(notifications WHERE is_read = FALSE AND recipient_id = current_user.id)

## 3.10 STUDENT PROMOTION RULES

  Promotion condition: Annual Average % >= 50%
  If >= 50% → PROMOTED to next class
  If < 50% → REPEATS current class

  Class progression order:
    NURSERY 1 → NURSERY 2 → NURSERY 3 → PRIMARY 1 → PRIMARY 2
    → PRIMARY 3 → PRIMARY 4 → PRIMARY 5 → GRADUATED

  Primary 5 graduates → status = 'Graduated', archived next year

  Promotion process:
    1. Admin selects "from year" and "to year"
    2. System shows preview of each student's status and action
    3. Admin confirms
    4. FOR EACH student:
       - Calculate annual % (see formula section)
       - If promoted: UPDATE students SET class_id = next_class_id
       - If repeated: class_id stays the same
       - If graduated: UPDATE students SET status = 'Graduated'
       - INSERT into student_promotion_records
    5. INSERT into student_promotions (summary)
    6. Log action

  Rollback: possible within same session (before next promotion batch)

## 3.12 BACKUP & RESTORE RULES

  Manual backup:
    - Fetch all data from all tables via API
    - Package as JSON
    - Download as file: backup_ECOLE_YYYYMMDD.json

  Restore:
    - Upload JSON backup file
    - Show warning: "This will DELETE all current data"
    - Require confirmation
    - DELETE all rows from all tables (in correct FK order)
    - INSERT all data from JSON
    - Reload page

## 4.1 PHASE DETECTION

  phase = (today < term.midterm_date) ? 'pre_midterm' : 'post_midterm'

---

## 4.2 PRE-MIDTERM CALCULATIONS

### For Nursery (French) — Pre-Midterm:
  Input: Raw scores from quizzes/tests (not Exam type)
         Each score out of its assessment.max_marks
  Output: NOTE (out of subject.mg_max, typically 50)

  NOTE = average(raw_scores) scaled to mg_max
       = (sum(raw_scores) / sum(max_marks)) × mg_max

  Example:
    Quiz1 = 48/50, Quiz2 = 46/50, Midterm = 49/50
    NOTE = ((48+46+49) / (50+50+50)) × 50 = (143/150) × 50 = 47.67

  COTE = getGrade(NOTE / mg_max × 100)
    e.g. 47.67/50 = 95.3% → A+

### For Primary (English) — Pre-Midterm:
  Input: Raw scores from quizzes/midterm (not Exam type)
  Output: SCORE (percentage, 0-100)

  SCORE = (sum(raw_scores) / sum(max_marks)) × 100

  Example:
    Quiz1=38/50, Quiz2=41/50, Quiz3=36/50, Midterm=42/50
    SCORE = (38+41+36+42)/(50+50+50+50) × 100 = (157/200) × 100 = 78.5%

  GRADE = getGrade(SCORE)

### Total for Pre-Midterm:
  totalMaxPossible = SUM over all subjects_with_assessments (subject.mg_max for Nursery, or 100 for Primary)

  IMPORTANT: Only subjects that have at least one assessment in this term
             contribute to totalMaxPossible. This is the key denominator rule.

  For Nursery:  overallPercent = (sum(all subject NOTEs) / totalMaxPossible) × 100
  For Primary:  overallPercent = (sum(all subject SCOREs) / (num_assessed_subjects × 100)) × 100
                                = sum(SCOREs) / num_assessed_subjects  [already %]
    Actually more accurately:
    overallPercent = (totalScore / totalMaxPossible) × 100
    where totalMaxPossible = count(subjects_with_assessments) × 100

---

## 4.3 POST-MIDTERM CALCULATIONS

### MG (Midterm Grade / Continuous Assessment):
  Input: Raw scores from assessment_type IN ('Quiz', 'Assignment', 'Observation', 'Midterm')
  Output: MG value (out of subject.mg_max)

  MG = (sum(mg_raw_scores) / sum(mg_max_marks)) × subject.mg_max

  Example (Mathematics, mg_max=50):
    Quiz1=45/50, Quiz2=42/50, Quiz3=48/50, Midterm=46/50
    MG = (45+42+48+46)/(50+50+50+50) × 50 = (181/200) × 50 = 45.25

  Special rule for post_midterm_only subjects:
    IF subject.appears_only_post_midterm = TRUE:
      MG = EX  (copied automatically)
      Teacher only enters EX
      MG = NULL → system sets MG = EX before calculation

### EX (Exam Grade):
  Input: Raw scores from assessment_type IN ('Exam', 'Final Exam')
  Output: EX value (out of subject.ex_max)

  EX = (sum(ex_raw_scores) / sum(ex_max_marks)) × subject.ex_max

  Example (Mathematics, ex_max=50):
    Exam1=44/50, Final=48/50
    EX = (44+48)/(50+50) × 50 = (92/100) × 50 = 46.0

### TOT (Total per subject):
  TOT = MG + EX

  If only MG exists (no exam yet): TOT = MG
  If only EX exists (no quizzes yet): TOT = EX
  If neither: TOT = NULL

### Overall Post-Midterm:
  totalMaxPossible = SUM over subjects_with_assessments (subject.mg_max + subject.ex_max)

  TOT_MG = SUM over all subjects (MG)
  TOT_EX = SUM over all subjects (EX)
  G_TOT  = TOT_MG + TOT_EX

  overallPercent = (G_TOT / totalMaxPossible) × 100

  IMPORTANT: Same denominator rule applies — only subjects with assessments count.

---

## 4.4 ANNUAL CALCULATIONS

### Per subject across all 3 terms:
  Annual_Tot_MG (subject) = Term1_MG + Term2_MG + Term3_MG
  Annual_Tot_EX (subject) = Term1_EX + Term2_EX + Term3_EX
  Annual_G_TOT  (subject) = Annual_Tot_MG + Annual_Tot_EX

  Annual max per subject = (mg_max + ex_max) × 3  [if subject in all 3 terms]

### Overall annual totals:
  ANNUAL_TOT_MG = SUM over all subjects (Annual_Tot_MG)
  ANNUAL_TOT_EX = SUM over all subjects (Annual_Tot_EX)
  ANNUAL_G_TOT  = ANNUAL_TOT_MG + ANNUAL_TOT_EX

### Annual max calculations:
  Primary annual max:
    Math/English/Kinyarwanda/French: (50+50)×3 = 300 each × 4 = 1200
    SET/SRS:                         (40+40)×3 = 240 each × 2 = 480
    Reading/Creative Arts:           (20+20)×3 = 120 each × 2 = 240
    Sports:                          (10+10)×3 = 60           = 60
    Total Primary Annual MAX = 1980

  Nursery annual max:
    8 subjects × (50+50) × 3 = 2400

### Annual percentage:
  Primary:  annualPercent = (ANNUAL_G_TOT / 1980) × 100
  Nursery:  annualPercent = (ANNUAL_G_TOT / 2400) × 100

### Annual grade:
  annualGrade = getGrade(annualPercent)

---

## 4.5 GRADING FUNCTION

  getGrade(percent):
    Look up grading_scale table:
      WHERE min_percent <= percent <= max_percent
      RETURN grade label

  Default:
    A+ = 90-100
    A  = 80-89
    B  = 70-79
    C  = 60-69
    D  = 50-59
    F  =  0-49

---

## 4.6 RANKING FORMULA

  For a class in a given term/phase:
  1. Calculate overallPercent for each student
  2. Sort students: ORDER BY overallPercent DESC, full_name ASC
     (ties broken alphabetically by full name A→Z)
  3. Assign rank: 1, 2, 3, ...
     (no shared ranks — tie broken by name)
  4. Display as "Xth of Y" where Y = total students in that list

---

## 4.7 FEE BALANCE FORMULA

  Total Fees = SUM(student_fees.amount WHERE student_id=X AND is_waived=FALSE)
  Total Paid = SUM(student_fees.paid_amount WHERE student_id=X)
  Total Waived = SUM(student_fees.amount WHERE student_id=X AND is_waived=TRUE)
  Balance = Total Fees - Total Paid
  Credit = student_credit_balance.credit_amount

  Collection Rate (per student) = (Total Paid / Total Fees) × 100
  Collection Rate (per class)   = SUM(Paid) / SUM(Fees) for all students in class × 100
  Collection Rate (school-wide) = SUM(All Paid) / SUM(All Fees) × 100

---

## 4.8 OVERDUE CALCULATION

  Days Overdue = MAX(0, today - student_fees.due_date)

  Severity:
    Critical: Days >= 30
    Warning:  Days 15-29
    Mild:     Days 7-14
    Recent:   Days 1-6
    Not yet:  Days <= 0

---

## 4.9 TERM PROGRESS

  Progress % = (today - term.start_date) / (term.end_date - term.start_date) × 100
  Days Remaining = term.end_date - today
  Days Into Term = today - term.start_date

---

## 4.10 PROMOTION FORMULA

  Condition: annualPercent >= 50  → PROMOTED
             annualPercent < 50   → REPEAT

  Exception: No marks at all → cannot promote automatically, admin decides

---

## 4.11 CLASS REGISTER: DENOMINATOR RULE (CRITICAL)

  The correct way to calculate percentage in class register:

  totalMaxPossible = SUM over subjects where subject HAS at least one assessment
                     in this class AND this term
                     × appropriate max marks

  This means:
    - If only 3 out of 9 subjects have been assessed,
      totalMaxPossible = max marks of those 3 subjects only
    - All students are compared against the same totalMaxPossible
    - A student who scored 0 on an assessed subject is not excluded
    - A subject with no assessments is not included in the denominator

  This ensures fair comparison — everyone is measured against the same ceiling.

# PART 9: STATISTICS & ANALYTICS CALCULATIONS
# ════════════════════════════════════════════════════════════

## Teacher Performance:
  class_average = AVG(overallPercent) for all students in class for term
  completion_rate = count(marks entered) / count(expected marks) × 100
  on_time_rate = count(assessments submitted by due_date) / count(total) × 100

## School-wide statistics:
  pass_rate = count(students WHERE overallPercent >= 50) / total_students × 100
  grade_distribution = count per grade level (A+, A, B, C, D, F)
  class_ranking = classes sorted by average overallPercent DESC
  subject_averages = per subject across all classes and terms

## Finance analytics:
  total_expected = SUM(fee_amounts per class × students per class)
  total_collected = SUM(payments) 
  collection_rate = total_collected / total_expected × 100
  by_method = SUM(amount) GROUP BY payment_method
  monthly_trend = SUM(amount) GROUP BY YEAR_MONTH(payment_date)
